/**
 * TEST: User logged in - view cart with items
 * Expected: Display cart with all user info
 */
GIVEN user is logged in with id=1
AND user has 3 items in cart
WHEN GET /cart
THEN

  - currentUser.getId() == 1 ✅

  - currentUser.getFullName() == "John Doe" ✅
  - cartItems.size() == 3 ✅
  - subtotal > 0 ✅

  - Model has 'currentUser' attribute ✅
  - Response returns "layout/main" ✅

 * TEST: User not logged in - redirect to login

  - No NullPointerException ✅

 * TEST: User logged in - cart is empty

// ========================================
// 9. INTEGRATION TESTS
// ========================================

/**
 * TEST: Full user flow
 */
SCENARIO: Complete shopping cart flow

1️⃣  User registers
    POST /api/auth/register
    → User created in database

2️⃣  User logs in
    POST /api/auth/login
    → CustomUserDetails created with full user info
    → Authentication token issued

3️⃣  User views cart
    GET /cart with @AuthenticationPrincipal CustomUserDetails
    → currentUser loaded with id, email, fullName, roles
    → Cart displayed with user info

4️⃣  User adds item
    POST /api/cart/add (or via product page)
    → Item added to cart

5️⃣  User updates quantity
    POST /cart/update/5?quantity=3
    → currentUser not null ✅
    → Item updated in database ✅
    → Redirect to /cart with success message ✅

6️⃣  User removes item
    GET /cart/remove/5
    → Item removed from cart
    → Success message displayed

7️⃣  User clears cart
    GET /cart/clear
    → All items removed
    → Cart is empty

8️⃣  User logs out
    → Authentication cleared
    → Next /cart request redirects to /login
